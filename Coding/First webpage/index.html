How a form works:

  1. The user fills in form data and presses a button to submit the information to the server.
  2. The name of each form control is sent over to the server, along with the value of the users entries.
  3. The server processes the information with a programming language such as PHP, C#, VB.net, or Java. It may also store the entered information in a database for security and data keeping reasons.
  4. Lastly, the server careates a webpage to send back to the browser based on the useres selected options or information recieved.

In order to differentiate between various pieces of information which may be contained in a form, information is sent to the server in a Name/Value pairs. If the form is one which allows the user to enter text, the the value of the form control is whatever the user has typed in. If the form control allows you to choose from a fixed set of answers (e.g. radio buttons, checkboxes, and drop-down boxes) then, the webpage author will add code that gives each option an automatic value. (Never change the name of a form control on a webpage unless you know the changed name is coded into the website previously as an option.)

Form Structure

The <form> element is used to specify a form and is always used with the action attribute and will usually have a method and id attribute as well. Every <form> elemt must carry an action attribute as previously stated. The value of the action attribute is the URL on the server of which you would like the information submitted to.

Forms can be sent using one of the two methods: get or post. With the get method, the values of the form are added to the end of the URL specified in the action attribute (which is also where you would like the information submitted sent to.) The get method is best for:

  • Short forms (such as search boxes, emails, names etc.)
  • When you are using the form strictly to recieve data from the web server. (Not for sending information to a database that should be added or deleted from the database.)

The post method are sent in what are known as HTTP headers. As a rule of thumb, you should use the post method for the following reasons and or applications:

  • The post method allows users to upload a file.
  • If the form is very long, the post method should be used.
  • If the form contains sensitive data, such as passwords or other protected personal information.
  • If the form is being utilized to add or remove information from a database, then the post method should be used.

Introducing CSS

Understanding CSS:

The easiest way to begin to understand CSS and how it works is to imagine that every element, (block or inline) element has a box around it specifying what should be styled. Here are some example styles of what can be styled within each type of element:

  • Boxes
    • Width and Height
    • Borders (Colors, width, and style).
    • Background color and images.
    • Position in the browser window.
  • Text
    • Typeface
    • Size
    • Color
    • Italics, bold, uppercase, lowercase, small-caps
  • Specific
    • Tables
    • Graphs
    • Lists
    • Forms

CSS allows you to set parameters for multiple elements within one line of code. In other words, CSS rules are applied to different HTML elements as a whole in order to style them. Below is a beginner example of how CSS code is written in order to specify and edit inline and block elements:

body { font-family: Arial, Verdana, Sans-Serif;}

h1, h2, h4 { color: #2AE9D6;}

p { color: #EF0E0E;}

CSS code contains two seperate parts that makeup the code itself. A selector and declaration the selector indicates which element the rule applies to. The same rule can be applied to more than one element if you seperate the element names with commas as seen above. The declaration indicates how the elements referred to in the selector should be treated and styled. Declarations are split into two parts, property and value, which are seperated by a comma.

Looking at the above code, the property is the word color and the value is the actual color code #2AE9D6.

Using External CSS

In order to use external CSS, a <link> element must be used to tell the browser where to find the CSS styling for that specific webpage. The <link> element is an empty element, meaning it does not have a closing tag. It also lives inside the <head> element. The <link> element should contain three seperate attributes:

href
This specifies the path to the CSS file. (Often in a file named CSS or Styles)
type
This attribute specifies the type of file being linked to. (The value should always be text/css for a CSS file.)
rel
This dictates the relation between the HTML page and the file it is being linked to. The value should be stylesheet when linking to a CSS file.

Your HTML page can include multiple CSS style sheets. To do this, it could have a <link> element for every CSS file it is utilizing. Some authors may use one CSS file to control the presentation (such as fonts and colors) and another CSS file to control the layout of the page.

Using Internal CSS

You can add CSS rules directly to your page by placing them inside a <style> element which typically lives inside of a <head> element of a page. The <style> element should use the type attribute to specify that the styles are being specified with CSS, the value should be text/css

When building a site with more than one webpage, you should always use an external CSS style sheet for the following reasons.

  1. It allows all pages to use the same style rules, rather than having to write out the code again seperately in every page.
  2. You can keep the content seperate from how the page looks.
  3. You can easily revise or edit all the content for all the pages at one time from one location rather than having to change every single page.

CSS Selectors

There are many selectors to allow you to specifically target any element in an HTML document. CSS selectors are case sensitive, so they must match the element names and attribute values exactly. There are more advanced selectors that allow you to select elements based on their attributes and values.

Meaning Example
Universal Selector Applies to all elements in a document. * {}
Targets all elements on the page.
Type Selector Matches element names. h1, h2, h3 {}
Targets the <h1>, <h2>, and <h3> elements.
Class Selector Matches an element whose class attribute has a value that matches the one specified after the period (or full stop) symbol. .note {}
Targets any element whose class attribute has a value of note (Or whatever class attribute you specify.)
p.note {}
Targets only <p> elements whose class attribute has a value of note.
ID Selector Matches an element whose id attribute has a value that matches the one specified after the pound or hash symbol. An example from this page would be as follows: #markup
Target the element whose id attribute has a value of markup
Child Selector Matches an element that is a direct child of another. li>a {}
Targets any <a> elements that are children of an <li> element (but not any other <a> elements on the page.)
Descendant Selector Matches an element that is a descendent of another specified element. (Not just a direct child of that element.) p a {}
In the example above, this code will target any <a> element inside of a <p> element, even if there are other elements nested between them.
Adjacent Sibling Selector Matches an element that is the next sibling of another. h1+p {}
In this example you are targetting the first <p> element after any <h1> element (but not other <p> elements)
General Sibling Selector Matches an element that is a sibling another, although it does not have to be the directly preceding element. h1~p {}
If you have two <p> elements that are siblings of an <h1> element, this rule would apply to both.

How CSS rules Cascade

If two selectors are identical, the second selector will take place over the first. If one selector is more specific than the others, the more specific rule will take precedence over more general ones. You can add the !important after any property value to indicate that it should be considered more important than other rules that apply to the same element.

Inhertitance

If you specifiy the font-family on the body element, they will apply to most child elements. This is because the value of the font-family is inherited by child elements. You can force a lot of properties to inherit values from their parent elements by using inherit for the value of the properites.
